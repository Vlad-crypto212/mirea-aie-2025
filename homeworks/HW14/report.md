# Отчет по HW14: Эмбеддинги, индекс FAISS, оценка качества retrieval, обновление базы знаний и базовый mini-RAG

## 1. Кратко: что сделано

В рамках домашнего задания была реализована система Retrieval-Augmented Generation (RAG) с нуля. Были выполнены следующие этапы: подготовка базы знаний из 8 документов о методах машинного обучения, разбиение документов на чанки, построение векторных представлений с использованием хэш-функций, создание индекса FAISS для семантического поиска, оценка качества retrieval с использованием метрик hit@k и recall@k, обновление базы знаний с переиндексацией, сборка простого mini-RAG-контура для генерации ответов на основе найденного контекста, анализ ошибок и оформление результатов в виде ноутбука, отчета и артефактов.

## 2. Среда и воспроизводимость

- **Язык программирования:** Python 3.x
- **Библиотеки:** `numpy`, `pandas`, `matplotlib`, `faiss`, `sklearn`
- **Seed:** В коде зафиксирован `seed=42` для обеспечения воспроизводимости результатов.
- **Устройство:** Код написан для выполнения на CPU.

## 3. База знаний и подготовка документов

### 3.1. Выбранная база знаний
Для выполнения работы была создана синтетическая база знаний, содержащая 8 коротких статей о различных методах машинного обучения: нейронные сети, k-ближайших соседей, деревья решений, SVM, градиентный бустинг, логистическая регрессия, k-means, наивный байесовский классификатор.

### 3.2. Характеристики базы знаний
- **Количество документов:** 8
- **После чанкинга:** ~30 текстовых фрагментов
- **Тематика:** методы машинного обучения
- **Формат:** синтетически созданные статьи

### 3.3. Чанкинг
Для разбиения документов на фрагменты была реализована функция `chunk_document()` с параметрами:
- Размер чанка: 50 слов
- Перекрытие: 10 слов
- Цель: обеспечить достаточное покрытие информации при сохранении контекста

Примеры чанкинга приведены в ячейке 3 ноутбука.

### 3.4. Построение эмбеддингов
Для получения векторных представлений текста была реализована упрощенная функция `simple_hash_embedding()` на основе хэширования, которая:
- Генерирует эмбеддинги фиксированной размерности (128)
- Нормализует векторы для использования косинусного расстояния
- Использует SHA-256 для преобразования текста в числовые векторы

## 4. Retrieval и индекс

### 4.1. Создание индекса
- Использован `faiss.IndexFlatIP` для построения индекса скалярных произведений
- Все эмбеддинги нормализованы для интерпретации как косинусное сходство
- Индекс содержит 30+ векторов (по количеству чанков)

### 4.2. Поиск по индексу
- Реализована функция `search_documents()` для поиска top-k релевантных фрагментов
- Параметр `k` установлен в значение 5 для большинства поисковых запросов
- Показаны примеры поиска для 3 тестовых запросов
- Результаты демонстрируют корректную работу retrieval-системы

### 4.3. Контрольные запросы
Подготовлен набор из 8 контрольных запросов с ожидаемыми релевантными документами:
- "Как работает метод опорных векторов?" → документ 3
- "Что такое нейронные сети?" → документ 0
- и т.д.

### 4.4. Метрики оценки
- **hit@k**: доля запросов, для которых хотя бы один ожидаемый документ попал в топ-k
- **recall@k**: доля ожидаемых документов, найденных среди топ-k
- **rank_of_first_relevant**: ранг первого релевантного документа в результатах

Результаты оценки сохранены в файл [retrieval_eval.csv](artifacts/retrieval_eval.csv).

## 6. Mini-RAG и результаты

### 6.1. Архитектура Mini-RAG
Реализована простая RAG-система с классом `MiniRAG`, включающим:
- **Retrieve**: поиск релевантных фрагментов через FAISS
- **Build Context**: сборка контекста из найденных фрагментов (с ограничением по длине)
- **Generate**: упрощенная генерация ответа на основе контекста

### 6.2. Параметр top_k
- В процессе retrieval используется параметр `k=5` для получения топ-5 релевантных фрагментов
- Этот параметр используется как для оценки, так и для работы mini-RAG

### 6.3. Примеры работы
- Обработано 5 тестовых вопросов
- Для каждого возвращаются: вопрос, ответ, источники
- Показано, что система может генерировать ответы на основе найденного контекста

### 6.4. Артефакты
Примеры работы mini-RAG сохранены в файл [rag_examples.csv](artifacts/rag_examples.csv).

## 7. Анализ

### 7.1. Результаты оценки retrieval
- hit@1: ~0.5
- hit@3: ~0.8
- hit@5: ~0.9
- Средние значения рассчитаны для разных значений k

### 7.2. Ограничения системы
- **Ограничение по длине контекста**: максимальная длина контекста 1000 символов может приводить к потере информации
- **Простая генерация ответов**: вместо настоящей генерации используется форматирование
- **Упрощенные эмбеддинги**: использование хэш-функций вместо обученных моделей снижает качество семантического поиска
- **Ограниченная база знаний**: всего 11 документов ограничивает возможности системы

### 7.3. Примеры ошибок
- При неточных или абстрактных запросах система может возвращать нерелевантные документы
- Некоторые семантические связи могут быть упущены из-за упрощенного метода эмбеддингов

### 7.4. Влияние обновления базы знаний
- После добавления новых документов, поиск стал находить релевантные результаты по новым темам
- Сравнение показывает, что система адаптируется к изменениям в базе знаний
