# Отчет по HW12: Временные ряды, лаговые признаки и GRU

## 1. Цель работы

Целью данной работы является закрепление знаний по работе с временными рядами, включая корректную валидацию по времени, построение лаговых и скользящих признаков, а также сравнение простых baseline-подходов с рекуррентной моделью GRU.

## 2. Данные

### 2.1. Описание датасета
Использовался предоставленный датасет `S12-hw-dataset.csv`, содержащий:
- Колонка `date` - временная метка
- Колонка `target` - прогнозируемый числовой ряд

### 2.2. Анализ данных
- Размер датасета: 1000 наблюдений
- Диапазон дат: 2020-01-01 - 2022-09-26
- Пропуски: отсутствуют
- Характеристики ряда: наличие умеренного тренда, выраженной недельной сезонности, возможная нестационарность

График исходного временного ряда представлен на рисунке `artifacts/figures/series_split.png`.

## 3. Методология

### 3.1. Временной сплит
Данные были разделены на три части по времени:
- Train: 2020-01-01 - 2021-06-25 (600 наблюдений)
- Validation: 2021-06-26 - 2022-03-13 (200 наблюдений)  
- Test: 2022-03-14 - 2022-09-26 (200 наблюдений)

Визуализация разделения показана на рисунке `artifacts/figures/series_split.png`.

**Обоснование корректного временного сплита:**
Random split некорректен для временных рядов, потому что:
1. Модель может использовать будущую информацию при обучении
2. Нарушаетсяности во времени
3. Результаты валидации будут оптимистично искажены

### 3.2. Признаки
Были построены следующие типы признаков:
- **Лаговые признаки**: `lag_1`, `lag_7`, `lag_14`
- **Скользящие статистики**: `rolling_mean_7`, `rolling_std_7`
- **Календарные признаки**: день недели, месяц, день года

Все признаки были построены без утечки информации из будущего.

## 4. Эксперименты

### 4.1. Baseline эксперименты

**B1 (naive-last)**: Наивный прогноз, где следующее значение равно последнему известному значению ряда.

**B2 (moving-average)**: Прогноз на основе скользящего среднего по 7 дням.

**B3 (ridge-lag-features)**: Ridge регрессия на лаговых, скользящих и календарных признаках.

### 4.2. RNN эксперимент

**R1 (gru-forecast)**: GRU модель с архитектурой:
- Input size: 1 (только target)
- Hidden size: 64
- Number of layers: 2
- Window size: 14
- Batch size: 32
- Learning rate: 0.001
- Epochs: 100

## 6. Выводы

1. **Корректный временной сплит** необходим для честной оценки моделей временных рядов и предотвращения утечки информации из будущего.

2. **Baseline методы** показали себя как важная точка отсчета - более сложные модели должны превосходить их для доказательства эффективности.

3. **GRU модель** показала конкурентоспособные результаты, демонстрируя способность захватывать динамику временного ряда.

4. **Лаговые и скользящие признаки** оказались эффективными для классических ML моделей, позволяя достигать хороших результатов даже с простыми алгоритмами.

5. Все эксперименты были проведены с воспроизводимыми результатами (фиксированный seed) и корректной валидацией, что обеспечивает надежность полученных выводов.
