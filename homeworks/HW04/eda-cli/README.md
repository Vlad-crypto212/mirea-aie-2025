# S04 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 04 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S04):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

Дополнительные возможности CLI (HW04)
Новые параметры команды report

Команда report теперь поддерживает дополнительные параметры:

-`max-hist-columns INTEGER`: Максимальное количество числовых колонок для включения в гистограммы (по умолчанию: 6).
-`top-k-categories INTEGER`: Количество топ-категорий для отображения в отчёте по категориальным признакам (по умолчанию: 10).
-`title TEXT` Заголовок для генерируемого отчёта (по умолчанию: EDA Report).
-`min-missing-share FLOAT`: Минимальная доля пропусков в столбце, при которой он будет отмечен как "проблемный" в отчёте (по умолчанию: 0.5).

Пример использования:
```bash
uv run eda-cli report data/example.csv --out-dir reports --max-hist-columns 5 --top-k-categories 20 --title "My Custom Report" --min-missing-share 0.3
```

Новые эвристики качества данных
В отчёт теперь включаются результаты новых эвристик, оценивающих качество данных:

-`has_constant_columns`: Проверяет, есть ли колонки, где все значения одинаковые.
-`has_high_cardinality_categoricals`: Проверяет, есть ли категориальные признаки с очень большим числом уникальных значений.
-`has_suspicious_id_duplicates`: Проверяет, что идентификаторы (столбцы, заканчивающиеся на _id) уникальны.
-`has_many_zero_values`: Проверяет, есть ли числовые колонки с долей нулей выше заданного порога.
Эти флаги и связанные с ними списки проблемных столбцов отображаются в разделе "Качество данных" генерируемого Markdown-отчёта.


HTTP-сервис (FastAPI) (HW04)
Проект был расширен HTTP-сервисом на базе FastAPI.
```bash
uv run uvicorn eda_cli.api:app --reload --port 8000
```

Сервис будет доступен по адресу http://127.0.0.1:8000

Документация к API доступна по http://127.0.0.1:8000/docs


Эндпоинты
-`GET /health`: Проверка состояния сервиса.
-`POST /quality`: Оценка качества по метрикам (n_rows, n_cols, missing_count).
-`POST /quality-from-csv`: Оценка качества по содержимому CSV-файла (возвращает quality_score).
-`POST /quality-flags-from-csv`: Возвращает полный набор флагов качества, включая флаги, добавленные в HW03 (например, `has_constant_columns`, `has_high_cardinality_categoricals`, `has_suspicious_id_duplicates`, `has_many_zero_values`). Принимает CSV-файл.
